API Abuse та CORS Misconfiguration

4. API Abuse

Вразливий API виконує дорогу операцію без обмежень. Захищений API має ліміт одночасних операцій і лічильник запитів.

Вразливий GET /api/security/abuse/vulnerable
Захищений GET /api/security/abuse/protected

5. CORS Misconfiguration

Вразливий API дозволяє читання з будь-якого origin. Захищений API дозволяє читання тільки довіреному frontend origin.

Вразливий GET /api/security/cors/vulnerable/profile
Захищений GET /api/security/cors/protected/profile

Оберіть демонстрацію.
